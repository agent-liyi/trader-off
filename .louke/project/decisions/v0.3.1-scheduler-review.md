# Decision: v0.3.1 Scheduler Migration Review

- **Date**: 2026-07-21
- **Spec**: v0.3.1-001-clock-rewind-scheduler-review
- **verdict**: (M) Migrate to `quantide.core.scheduler.SchedulerManager`

## Evidence

### (a) SchedulerManager API signatures

Read from `quantide/core/scheduler.py` (agent-liyi/millionaire):

| Method | Signature |
|--------|-----------|
| `init` | `(self, timezone=None)` |
| `start` | `(self)` |
| `stop` | `(self)` |
| `add_job` | `(self, func, trigger=None, args=None, kwargs=None, id=None, name=None, **trigger_args)` |
| `add_listener` | `(self, callback, mask=None)` |

Uses `BackgroundScheduler` (apscheduler) internally; decorated with `@singleton`.

### (b) trader-off scheduler callsites

`src/trader_off/scheduler/` module (11 files):
- `core.py` — RetrainScheduler with async start/stop/trigger_now/get_status (pure asyncio, no apscheduler)
- `cron.py` — CronTrigger / next_cron_fire (croniter backend)
- `cli.py` — CLI bindings (build_retrain_parser, run_scheduler_start/stop/status)
- `ports.py` — Protocol-based ClockPort/TrainerPort
- `deploy.py`, `perf_monitor.py`, `registry.py`, `drift/`, `state.py`, `api.py`

The scheduler currently uses croniter for cron scheduling and asyncio for the main loop. No apscheduler dependency exists in this module. The migration to SchedulerManager adds a new adapter (`adapter.py`) without modifying existing croniter paths.

### (c) Isolation impact assessment

- **v0.3.0 NFR-0100**: "module-level zero `import quantide`" — superseded by NFR-0101 for scheduler module only
- **NFR-0101**: function-scope lazy import + zero business symbol import
  - `from quantide.core.scheduler import scheduler` only in function bodies
  - No `quantide.service.*`, `quantide.data.*`, `quantide.portfolio.*` imports
  - AST validation confirms all quantide imports inside FunctionDef/AsyncFunctionDef
- **Testability**: function-scope imports are mockable via `patch()` at call time; module-level imports would require `sys.modules` manipulation
- **Static analysis**: grep for top-level `^import quantide\|^from quantide` in `src/trader_off/scheduler/` must return empty

## Conclusion

- **verdict = (M) Migrate** — create thin `QuantideSchedulerAdapter` in `scheduler/adapter.py`
- **Path**: function-scope `from quantide.core.scheduler import scheduler` → wrap as adapter
- **Side effects**: NFR-0100 → NFR-0101 relaxation; `history.md` updated; v0.4.0 backlog cleared of scheduler review item
- **No refactor**: existing `cron.py` / `cli.py` / `core.py` unchanged; adapter is additive only
