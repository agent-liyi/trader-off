Align trader-off's DataLoader with quantide's fetcher interface via a thin adapter. trader-off currently has its own DataLoader for loading market data. quantide (a separate data platform) defines a standard fetcher interface. We need to create a thin adapter that makes trader-off's DataLoader conform to quantide's fetcher contract, enabling seamless data consumption between the two systems. Key objectives: 1) Define the adapter layer that bridges DataLoader and quantide's fetcher interface 2) Ensure all DataLoader methods map cleanly to the fetcher contract 3) Handle date ranges, symbol filtering, and field selection consistently 4) Maintain backward compatibility. Out of scope: actual data fetching from external sources, tushare integration, network calls.
