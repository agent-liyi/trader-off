# v0.2.0 — 因子挖掘·再训练调度·组合优化器

## 概述

v0.2.0 在 v0.1.0 lightGBM 定价模型的基础上，将单次训练的静态模型升级为**持续进化的量化系统**。
三个核心模块环环相扣：因子挖掘提供更丰富的信号来源，再训练调度确保模型始终适应市场变化，
组合优化器将预测信号转化为风险可控的投资组合。

## 范围

### 1. 因子挖掘自动化 (Factor Mining Automation)

**目标**：从预定义因子模板库中系统化地生成、评估、选择有效因子，替代手工构造特征的过程。

**核心设计**：
- **因子模板库**：按类型组织的基础因子模板
  - 动量类：过去 N 日收益率、N 日超额收益、动量加速度等
  - 波动率类：N 日波动率、振幅比、ATR 等
  - 成交量类：成交量变化率、成交额占比、换手率偏离等
  - 基本面组合类：EP、BP、ROE、营收增速等（当基本面数据可用时）
- **表达式引擎**：支持模板参数化（窗口期 N、数据类型 close/open/high/low/volume 等），对合法组合进行枚举
- **评估流水线**：
  - 计算每个候选因子的 IC（Information Coefficient）和 ICIR（IC 均值/IC 标准差）
  - 支持 Rank IC（Spearman 秩相关）作为辅助指标
  - 输出因子评估报告：IC 时序、ICIR、分组收益、累计收益曲线
- **选择策略**：
  - Top-K 选择：按 ICIR 排序取前 K 个因子
  - 去冗余：Pearson 相关系数 > 0.9 的因子仅保留排名更高者
  - 可视化输出因子相关性热力图

**输出物**：
- 因子表达式注册表（YAML/JSON）
- 精选因子集（含权重）
- 因子评估报告（HTML / Markdown）

### 2. 模型再训练调度 (Model Retraining Scheduling)

**目标**：建立模型持续迭代和自动部署的调度机制，确保模型始终运行在最新市态上。

**核心设计**：
- **调度触发器（Scheduler）**：
  - Cron 定时：可配置周期（如每日/每周/每月收盘后触发）
  - 数据漂移检测：输入特征分布漂移（PSI / KS 检验）超过阈值时触发
  - 性能衰减检测：在线预测性能（回测 Sharpe / IC 衰减）超过阈值时触发
  - 手动触发：CLI / API 接口
- **重训范围**：
  - 全量重训（Full Retrain）：使用全部历史数据重新训练
  - 增量重训（Incremental Retrain）：在已有模型基础上，仅使用新数据进行更新（lightGBM`refit`）
  - 策略由配置决定，默认全量重训每 5 个交易日 + 增量重训每日
- **模型版本管理**：
  - 每次重训生成唯一版本号（`v{major}.{minor}.{build}`）
  - 版本化存储：模型文件 + 训练元信息（训练日期、数据范围、评估指标）
  - 回滚能力：支持按版本号切换线上模型
- **自动部署**：
  - 新模型通过验证后自动部署到预测服务
  - 保留前 N 个版本用于 A/B 比较或回滚

**输出物**：
- 调度器模块 + cron 集成
- 漂移检测器模块
- 模型版本管理器（版本注册表 + 存储/加载 API）
- 自动部署流水线

### 3. 组合优化器 (Portfolio Optimizer)

**目标**：将预测信号（因子合成得分或模型预测值）转化为满足约束的最优投资组合权重。

**核心设计**：
- **目标函数**：
  - 最大化 Sharpe Ratio（均值-方差框架）
  - 支持自定义风险厌恶系数 λ
- **约束条件**：
  - Long-only：所有资产权重 ≥ 0
  - 满仓约束：∑ w_i = 1
  - 行业中性：每个行业的权重偏离 ≤ 行业基准 ± δ
  - 个股集中度上限：单只个股权重 ≤ max_weight（默认 10%）
  - 换手率约束：单期换手率 ≤ turnover_limit（可选）
- **风险模型**：
  - 基础：样本协方差矩阵 → Ledoit-Wolf shrinkage 估计
  - 扩展：可使用因子风险模型（预留接口）
- **输入**：
  - 预期收益向量（来自预测模型或因子合成得分）
  - 协方差矩阵（历史收益或因子暴露）
  - 约束参数
  - 行业分类映射
- **输出**：
  - 最优权重向量
  - 组合诊断报告：预期 Sharpe、波动率、行业暴露、个股集中度、换手率
  - 有效前沿曲线（可选可视化）

## 非功能需求

### 性能预算 (P95 Latency)
- 单次训练全量流水线（含因子计算 + 训练）：≤ 300 秒
- 单次预测（全部候选因子 → 模型推理）：≤ 5 秒
- 单次回测（含组合优化）：≤ 600 秒
- 内存峰值：≤ 16 GB

### 质量要求 (DoD)
- 所有 e2e 测试通过
- 单元测试覆盖率 ≥ 97%
- Mutation testing 得分 ≥ 80%（mutmut）
- 性能预算满足 (train ≤ 300s, predict ≤ 5s, backtest ≤ 600s, mem ≤ 16GB)
- 文档同步检查通过（architecture/interfaces/docs sync + 至少 3 个 ADR 存放于 docs/adr/）
- 安全审计 (M-SECURITY) 通过
