# trader-off v0.1.0 — 基于 lightGBM 的短时 A 股定价模型

## 背景

trader-off 是一个基于 [millionaire](https://github.com/zillionare/millionaire) 量化框架的策略项目，目标是构建一个**短时资产定价模型（Short-horizon Asset Pricing Model）**，对 A 股个股进行未来 5 个交易日的收益率预测。

## 问题描述

A 股市场存在大量的定价异象和因子，传统多因子模型在短周期预测上表现有限。通过机器学习（特别是 lightGBM）对大量特征进行非线性建模，有望捕捉到更精细的定价信号。

## v0.1.0 目标（最小可行产品）

v0.1.0 的目标是搭建**端到端的机器学习预测管线**，并将其集成到 millionaire 的回测系统中，使策略能够基于模型信号进行交易并评估绩效。

### 范围包含

1. **特征工程模块**
   - 基于日线数据计算技术指标（动量、波动率、成交量相关等）
   - 支持扩展自定义特征
   - 特征标准化与缺失值处理

2. **训练管线**
   - 标签构建：未来 5 个交易日收益率
   - 训练数据：A 股全市场（约 4000+ 只股票）的日线数据
   - 模型：lightGBM 回归模型（L2 loss / 自定义 quantile loss）
   - 模型序列化与版本管理

3. **预测服务**
   - 加载已训练模型，对当前持仓/关注列表生成预测分数
   - 输出排序结果供交易决策使用

4. **millionaire 策略集成**
   - 编写继承 `quantide.core.strategy.BaseStrategy` 的策略类
   - 在 `on_day_close` 中调用模型，生成次日交易信号
   - 基于预测排序的简单交易规则（top/bottom 分位数买入/卖出）
   - 接入 millionaire 回测 Runner

5. **绩效分析面板**
   - 回测报告（年化收益、夏普比率、最大回撤等）
   - 预测能力分析（IC / Rank IC / 分层回测）
   - 特征重要性分析

### 不包含

- 实盘交易（v0.2+）
- 因子挖掘自动化（v0.2+）
- 分钟线/高频信号（v0.3+）
- 优化器/组合构建（v0.4+）

## 数据来源

- 通过 millionaire 框架的数据模块获取全 A 股日线行情数据
- 数据频率：日线（1d）
- 数据范围：全 A 股（含沪深主板、创业板、科创板）
- 数据字段：开盘、收盘、最高、最低、成交量、成交额、换手率等

## 技术栈

- Python ≥ 3.11
- lightGBM（预测模型）
- polars（数据处理）
- millionaire 框架（回测基础设施）
- loguru（日志）
- pytest（测试）
- uv（包管理）

## 用户故事

> 作为一名量化研究员，我希望能够基于 A 股日线数据训练一个 lightGBM 模型来预测个股未来 5 日收益率，并将预测信号接入 millionaire 回测系统，以便评估该模型在历史数据上的表现，并可视化回测结果与预测能力。
